Authentication Guide

Learn how to securely authenticate your requests to the Craft APIs.

Overview

The Craft API uses JSON Web Tokens (JWT) or API Keys for authentication. All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail with a 401 Unauthorized error.

Bearer Token

Include your API key or JWT token in the Authorization header of your HTTP requests. Our API expects the X-Craft-API-Key or standard Bearer format.

Header Format 
Authorization: Bearer <your_access_token>

Example Request

curl -X GET "https://api.craft.com/orders/v2/" \
  -H "X-Craft-API-Key: ck_live_YOUR_KEY_HERE" \
  -H "Content-Type: application/json"
import requests

url = "https://api.craft.com/orders/v2/"
headers = {
    "X-Craft-API-Key": "ck_live_YOUR_KEY_HERE",
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers)
print(response.json())
fetch("https://api.craft.com/orders/v2/", {
  headers: {
    "X-Craft-API-Key": "ck_live_YOUR_KEY_HERE",
    "Content-Type": "application/json"
  }
})
.then(response => response.json())
.then(data => console.log(data));

OAuth 2.0 Integration

For user-facing applications, use the OAuth 2.0 authorization code flow to obtain access tokens on behalf of users.

Authorization URL https://api.craft.com/oauth/authorize
Token URL https://api.craft.com/oauth/token

Common Errors

401 Unauthorized

The API key is missing, invalid, or expired.

{
  "error": "authentication_failed",
  "message": "Invalid or expired API key provided."
}

403 Forbidden

The authenticated key does not have the required scopes.

{
  "error": "insufficient_permissions",
  "message": "Key requires 'orders:write' scope."
}
